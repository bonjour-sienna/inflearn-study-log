## 오늘 배운 것 — 서브에이전트와 나만의 명령어, 하루 만에 만들기

`/agents` 대화형 마법사가 사라진 걸 발견한 게 시작이었다. "그럼 에이전트는 어떻게
만들지?"에서 출발해 유닛 테스트 에이전트 하나와 커스텀 슬래시 명령어 하나를 손에
넣기까지의 과정.

### ① 유닛 테스트 서브에이전트 만들기

에이전트는 **YAML frontmatter가 달린 마크다운 파일 하나**다. 마법사가 없어도 만드는
방법은 단순하다 — 무엇을 시킬지 말로 요청하면 된다.

내가 내린 요청은 이랬다:

> "유닛 테스트 에이전트를 만들어줘. **프로젝트 컨벤션·코딩 컨벤션·테스트 구조**를
> 신경 써서 만들어줘."

핵심 요구는 "테스트를 짜라"가 아니라 "**그 프로젝트의 기존 방식에 맞춰** 짜라"였다.
그래서 에이전트의 제1원칙을 *"쓰기 전에 먼저 조사하라"*(discover before you write)로
설계했다. 이 원칙이 시스템 프롬프트의 뼈대가 된다:

1. 매니페스트(package.json·pyproject.toml·go.mod)를 읽어 **테스트 러너와 어설션·모킹
   라이브러리**를 파악한다.
2. **기존 테스트 2~4개를 먼저 읽어** 파일 위치·네이밍·구조·모킹 스타일을 추출한다.
   CLAUDE.md나 테스트 문서가 있으면 그 규칙은 구속력 있는 것으로 취급한다.
3. 대상 모듈과 협력 객체를 읽고 **공개 동작**(입력·출력·부수효과·에러·경계)을 식별한다.

**파일 위치가 곧 스코프**다:

| 경로 | 스코프 |
|---|---|
| `~/.claude/agents/` | **user** — 모든 프로젝트에서 사용 |
| `./.claude/agents/` | **project** — 해당 저장소에서만 |

모든 프로젝트에서 쓰고 싶어 전역에 뒀다 → `~/.claude/agents/unit-test-writer.md`

frontmatter에 이름·설명·도구·모델을, 본문에 시스템 프롬프트를 담는다:

```markdown
---
name: unit-test-writer
description: >-
  Writes unit tests for existing code by first discovering the project's own
  conventions ... Use when the user asks to "write tests", "add unit tests" ...

  <example>
  user: "Add unit tests for the new discount calculator in pricing.ts"
  assistant: "I'll use the unit-test-writer agent to inspect the existing test
  setup and write tests that match your conventions."
  </example>
tools: Read, Write, Edit, Grep, Glob, Bash
model: sonnet
---

You are a senior test engineer. Your job is to write unit tests that look like
they were written by the team that owns this codebase ...
```

- `description`의 **`<example>` 블록은 "언제 이 에이전트를 자동으로 부를지" 힌트**가 된다.
  단순 설명이 아니라 라우팅 신호다.
- `tools:`는 **허용 권한 화이트리스트**다. 명시하면 "이것만", 생략하면 "전부 상속".
  `Read·Grep·Glob`(조사) + `Write·Edit`(작성) + `Bash`(테스트 실행)만 주고 웹 접근이나
  다른 에이전트 호출은 일부러 뺐다 — **최소 권한 원칙**.
- `model: sonnet` 한 줄이면 세션 모델과 무관하게 항상 그 모델로 실행. 지우면 다시 상속.

### ② `/agents` 마법사는 왜 사라졌나

**Claude Code v2.1.198 (2026-07-01)** 에서 제거됐다. 체인지로그의 표현은
*"Removed the `/agents` wizard; ask Claude to create or manage subagents, or edit
`.claude/agents/` directly."*

공식 사유는 **단순화(simplification)** — 터미널 마법사 UI를 없애고 **자연어 대화로
에이전트를 만들고 관리**하는 방식으로 옮겼다. 기능이 사라진 게 아니라 "터미널 마법사"만
사라진 것이다.

그래서 조회에도 전용 `list` 명령이 없다. 대신 프롬프트에 `@agent-` 를 입력하면 자동완성
메뉴에 목록이 뜬다. 폴더를 직접 보거나 `/doctor`(v2.1.205+)로 중복 파일을 점검할 수도 있다.

> ⚠️ **함정 — 만들었는데 `@agent-`에 안 뜬다?**
> Claude Code는 에이전트 폴더를 자동 감시하지만 **새 폴더에 그 스코프의 첫 에이전트를
> 만들 때는 예외**다. 이번처럼 `~/.claude/agents/`가 새로 생긴 경우 실행 중인 세션이
> 아직 감시하지 못한다. → **재시작하면** 시작 시 폴더를 스캔해 인식한다. 이 폴더에
> 추가하는 **두 번째부터는** 재시작 없이 몇 초 안에 잡힌다.

### ③ 조회용 커스텀 슬래시 명령어 만들기

"에이전트 목록을 `/agents`로 보고 싶다"에서 출발했지만 함정이 있었다.

- `/agents`는 **예약된 내장 명령어** — 같은 이름의 커스텀 명령은 내장에 가려져 덮어쓸 수 없다.
- 그래서 다른 이름으로. 처음엔 `/my-agents`를 제안했다가 최종적으로 **`/agent-list`** 로 확정.

커스텀 슬래시 명령어는 `commands/` 폴더에 산다 → `~/.claude/commands/agent-list.md`

명령어 파일도 **frontmatter + 본문** 구조다. 본문은 "실행됐을 때 Claude에게 시킬
프롬프트"이고, `allowed-tools`로 사용할 도구를 미리 허용해 둔다:

```markdown
---
description: List my custom subagents (name, scope, model, tools, description)
allowed-tools: Bash(ls:*), Bash(cat:*), Bash(sed:*), Bash(awk:*), Bash(echo:*), Read, Glob
---
List every custom subagent defined for this user and this project, then present
them in a single Markdown table.

Look in both locations:
- ~/.claude/agents/  (and subfolders) — scope: user (all projects)
- ./.claude/agents/  (and subfolders) — scope: project (this repo only)

For each *.md agent file, read its YAML frontmatter and extract:
- name / model (if omitted, show `inherit`) / tools (if omitted, show `all`)
- description (first sentence only)

Render a table: Agent | Scope | Model | Tools | Description.
```

실행하면 이런 표가 나온다:

| Agent | Scope | Model | Tools | Description |
|-------|-------|-------|-------|-------------|
| `unit-test-writer` | user | sonnet | Read, Write, Edit, Grep, Glob, Bash | 기존 컨벤션에 맞춰 유닛 테스트 작성 |

> 🔄 `commands/` 폴더도 새로 생겼으므로 슬래시 메뉴에 `/agent-list`가 뜨려면 **한 번의
> 재시작**이 필요하다. (에이전트 폴더와 같은 원리)

## 핵심 정리 — 치트 시트

| 무엇 | 어디에 | 만드는 법 |
|------|--------|-----------|
| **서브에이전트** | `~/.claude/agents/*.md` | 말로 요청 → frontmatter(name·description·tools·model) + 시스템 프롬프트 |
| **슬래시 명령어** | `~/.claude/commands/*.md` | frontmatter(description·allowed-tools) + 실행 프롬프트 |
| **조회** | — | `@agent-` 자동완성 · 폴더 확인 · 커스텀 `/agent-list` |
| **주의** | — | 새 폴더의 *첫* 파일은 재시작 후 인식 · 내장 명령어 이름은 덮어쓸 수 없음 |

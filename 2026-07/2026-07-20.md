## 오늘 배운 것 — 루프(/loop)와 루틴(Routines)

> Claude Code의 자동화 기능 두 가지. 2026년 상반기 출시 (루틴은 연구 프리뷰 중)

## 0. 한 줄 요약

| 기능 | 설명 |
|---|---|
| **루프 (/loop)** | 현재 세션 안에서, 내 컴퓨터에서 일정 간격마다 프롬프트를 반복 실행 |
| **루틴 (Routines)** | 프롬프트·저장소·커넥터를 저장해두면 Anthropic 클라우드에서 자동 실행 (컴퓨터 꺼도 동작) |

---

## 1. 루프의 4가지 유형

1. **턴 기반**: 일반 대화. 프롬프트마다 "컨텍스트 수집 → 작업 → 검증 → 응답" 사이클 반복
2. **목표 기반 (`/goal`)**: 완료 조건 정의 → 조건 달성 또는 최대 턴 도달까지 계속 작업
   - 예: `/goal Lighthouse 점수 90 이상, 5번 시도 후 중지`
3. **시간 기반 (`/loop`, `/schedule`)**: 정해진 간격마다 재실행
4. **프로액티브**: 위 요소들 + auto mode + dynamic workflows 조합, 사람 개입 없이 이벤트/스케줄로 실행

---

## 2. /loop 명령어

```
/loop 5m PR 확인하고 리뷰 코멘트 반영, CI 실패 수정
/loop 20m /review-pr 1234        # 스킬/명령도 반복 가능
```

- 간격 단위: `s`, `m`, `h`, `d` (최소 1분, 초는 올림)
- 간격 생략 시 Claude가 1분~1시간 사이에서 자동 조절
- **로컬 · 세션 한정**: 터미널 닫거나 컴퓨터 끄면 중단, 세션당 최대 50개
- 용도: 배포 모니터링, PR 감시, CI 확인 등 작업 중 잠깐의 반복 감시

---

## 3. 루틴 (Routines)

- **구성**: 프롬프트 + GitHub 저장소 + 커넥터를 하나로 패키징한 저장된 설정
- **실행 위치**: Anthropic 클라우드 → 노트북 꺼도 계속 작동

### 3-1. 트리거 3종 (조합 가능)

| 트리거 | 설명 |
|---|---|
| 스케줄 | 매시간/매일/평일/매주 또는 일회성 (최소 간격 1시간) |
| API | 루틴별 전용 엔드포인트에 POST → 즉시 실행 (알림·배포 파이프라인 연동) |
| GitHub | PR·릴리스 이벤트에 반응, 작성자·브랜치·라벨 필터 가능 |

### 3-2. 만드는 법

- 웹: `claude.ai/code/routines`
- CLI: `/schedule daily PR review at 9am` (별칭 `/routines`)
- 데스크톱 앱: Routines 메뉴

### 3-3. 주의사항

- 실행 중 권한 확인 없이 **완전 자율 실행** → 저장소·커넥터·네트워크 범위를 최소한으로 제한할 것
- 기본적으로 `claude/` 접두사 브랜치에만 푸시 가능
- 커밋·PR·Slack 메시지 등 모든 행동이 내 계정 명의로 표시됨
- Pro/Max/Team/Enterprise 요금제 + Claude Code on the web 활성화 필요
- 구독 사용량 차감 + 계정당 일일 실행 횟수 상한 있음

### 3-4. 활용 예시

- 매일 밤 이슈 백로그 정리 후 Slack 요약
- 모니터링 알림 발생 시 원인 분석 + 수정 PR 초안 생성
- 새 PR마다 팀 체크리스트 기반 자동 리뷰
- 배포 후 스모크 테스트로 go/no-go 판정
